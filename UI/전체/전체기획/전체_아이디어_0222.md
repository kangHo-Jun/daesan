# 상세_아이디어_0222

> 대상: 건축자재 쇼핑몰(B2B 리모델링업자)  
> 적용 위치: Cafe24 상품상세 `<div id="prdDetail" class="productDetail">` 내부에 **단일 HTML**로 삽입  
> 데이터 소스: **Google Sheet(내부 DB)**  
> 목표: **자동 생성 + 브랜드 일관성 + (Hover 그래프/스크롤 등장) 인터랙션 유지**

---

## 1. 결론 요약

- **상세 전체를 Satori(이미지)로 렌더링**하면  
  - 표/텍스트 **복사 불가**, SEO 약화, 접근성 저하, 모바일 가독성 이슈  
  - 그리고 **Hover 그래프 / 스크롤 등장** 같은 다이나믹 기능이 거의 불가
- 따라서 **Hybrid(권장)**:
  - 레이아웃·텍스트·표 → **HTML**
  - “그래프/만화/요약 인포카드/구조 다이어그램” → **Satori 이미지**

---

## 2. 섹션 구성(요구사항 반영)

### 2.1 섹션 4요소(상단 → 하단)

1) **회사 차별점** (한 줄 배너)  
2) **가격 동향** (일주일 상승/하락 + *가격 hover 시 그래프 팝업*)  
3) **만화 4컷** (2×2 홀더)  
4) **제품 상세** (좌우 2단)

> 제품 상세는 “텍스트/표” 비중이 높은 경우가 많아 **HTML 유지가 효율적**.

## 3. Hybrid 설계안(권장)

### 3.1 무엇을 Satori로 만들까?

| 요소 | Satori 권장 여부 | 이유 |
|---|---:|---|
| 가격 그래프(7일) | ✅ | 시각 카드화 효과 큼, HTML/CSS 간섭 최소화 |

## 4. 인터랙션 설계

### 4.1 가격 Hover 시 그래프 표시

**구조(HTML 컨테이너 + 그래프 이미지만 Satori):**
```html
<div class="zart-price">
  <span class="zart-price-label">이번주</span>
  <span class="zart-price-value">18,500원</span>

  <div class="zart-price-pop">
    <img src="__SATORI_GRAPH_URL__" alt="최근 7일 가격 동향 그래프">
  </div>
</div>
```

**CSS(순수 hover):**
```css
.zart-price { position: relative; display: inline-flex; gap: 8px; align-items: baseline; }
.zart-price-pop {
  position: absolute; left: 0; top: calc(100% + 10px);
  width: min(320px, 80vw);
  opacity: 0; transform: translateY(8px);
  pointer-events: none;
  transition: opacity .18s ease, transform .18s ease;
  background: #fff; border: 1px solid rgba(0,0,0,.08); border-radius: 6px;
  padding: 10px; box-shadow: 0 8px 24px rgba(0,0,0,.12);
  z-index: 10;
}
.zart-price:hover .zart-price-pop { opacity: 1; transform: translateY(0); }
```

> 모바일에서는 hover가 약하므로, **터치 시 토글(JS)** 옵션도 고려.

---

### 4.2 스크롤 시 섹션이 “동적으로 나타나게”

**구조:**
```html
<section class="zart-sec" data-animate="fade-up">...</section>
```

**CSS:**
```css
[data-animate] { opacity: 0; transform: translateY(18px);
  transition: opacity .5s cubic-bezier(.2,.8,.2,1), transform .5s cubic-bezier(.2,.8,.2,1);
}
[data-animate].is-visible { opacity: 1; transform: translateY(0); }
@media (prefers-reduced-motion: reduce) {
  [data-animate] { opacity: 1; transform: none; transition: none; }
}
```

**JS(IntersectionObserver):**
```html
<script>
(function(){
  'use strict';
  function init(){
    var els = document.querySelectorAll('[data-animate]');
    if(!('IntersectionObserver' in window) || !els.length){
      els.forEach(function(el){ el.classList.add('is-visible'); });
      return;
    }
    var io = new IntersectionObserver(function(entries){
      entries.forEach(function(e){
        if(e.isIntersecting){
          e.target.classList.add('is-visible');
          io.unobserve(e.target);
        }
      });
    }, { root: null, rootMargin: '-80px 0px', threshold: 0.12 });
    els.forEach(function(el){ io.observe(el); });
  }
  if(document.readyState === 'loading') document.addEventListener('DOMContentLoaded', init);
  else init();
})();
</script>
```

| comic_1~4_url | https://... | 4컷 이미지 |

| detail_right_md | ... | 우측 콘텐츠(마크다운) |

## 6. 생성 파이프라인 제안

### 6.1 최종 산출물

- `Cafe24_prdDetail_{product_code}.html`  
  - `<style> + <div> + <script>` 한 파일
  - `zart-` prefix로 네임스페이스 고정

### 6.2 흐름(권장)

1) Google Sheet에서 제품 데이터 읽기  
2) (옵션) 그래프/만화/카드 → Satori로 이미지 생성 후 URL 저장  
3) 템플릿 엔진으로 **HTML 조립(문자열 빌드)**  
4) 결과를 파일로 저장/배포(복붙 또는 자동 삽입 워크플로우)

## 8. 운영 체크리스트(Cafe24 제약 대응)

- [ ] 외부 `script src` / `link href` **사용하지 않기** (단일 HTML)  
- [ ] CSS/JS 전역 오염 방지: `zart-` prefix + IIFE  
- [ ] 이미지 URL은 https, 만료 없는 저장소(Cloudflare R2/S3/Drive 공개 등)  
- [ ] 모바일 hover 대체: “터치 토글” 옵션 고려  
- [ ] 테이블/텍스트는 HTML로 유지(복사/가독성/SEO)
