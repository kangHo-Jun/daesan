™## **1\. 디자인 철학 및 목표**

* **웹 UI 전용 컴포넌트 실사** (Compact Module)
* 포스터나 광고지가 아닌, 실제 브라우저의 "확장된 카드 블록"처럼 렌더링할 것
* 한글 폰트의 정밀한 렌더링 및 고밀도 정보 배치 필수
* **주의: 지시어(예: SECTION, Layout) 및 프롬프트 내의 설명 텍스트를 절대로 이미지에 포함하지 말 것.**

## **2\. 기술 규격 및 스타일 가이드**

* **캔버스:** 800 x 950 px 
* **배경:** #FFFFFF (Pure White)
* **타이포그래피 (v3 대비 25% 축소 사양):**
  * 강조 텍스트(주요 타이틀): 24-26px (NotoSansKR-Bold)
  * 강조 데이터(수치): 20-22px (NotoSansKR-Bold)
  * 일반 텍스트(라벨/설명): 13-15px (NotoSansKR-Regular)
* **밀도 및 여백:** 
  * 모든 수직 간격을 최소화하여 "정보 덩어리"가 낭비되는 공간 없이 촘촘하게 배치되도록 함
* **비주얼 스타일:**
  * **그림자:** 거의 보이지 않을 정도의 극도로 미세한 깊이감만 부여 (Shadow Offset < 2px)
  * **도넛 차트:** 선의 굵기를 기존의 70% 수준으로 얇고 모던하게 표현
  * **뱃지 스케일:** 정보 전달에 방해되지 않도록 전체 크기를 소폭 축소

## **3\. 시각적 블록 구성 (Top to Bottom)**

* **첫 번째 상단 영역 (제품 핵심):**
  * 제품의 실사 이미지가 좌측에, 제품명과 주요 스펙 및 뱃지가 우측에 배치되는 구조. 
  * 제목과 수치의 크기 균형을 웹 카드 상단 헤더 수준으로 조절함.

* **두 번째 중앙 영역 (기술 데이터):**
  * 좌측에는 제품 간의 단면 비교 비주얼을, 우측에는 정교한 수치 차트(얇은 라인 스타일)를 배치.
  * 불필요한 장식 요소를 배제하고 데이터 그 자체의 가독성에 집중함.

* **세 번째 하단 영역 (활용 가이드):**
  * 화이트 톤의 3개 카드가 가로로 나열되는 구조. 
  * 카드 간 여백을 좁게 설정하여 데이터 밀도를 높임.

## **5\. 최종 시스템 명령 (System Command)**

1. **캔버스에 그리는 모든 텍스트는 반드시 제공된 JSON 데이터 내의 값(`data` 객체)만을 사용할 것.**
2. 이 프롬프트 문서에 기술된 설명용 텍스트(예: "첫 번째 상단 영역", "디자인 철학", "v4")를 절대로 그리지 말 것.
3. ┌──┐ 와 같은 레이아웃 도식이나 "Layout 구조" 같은 메타 텍스트를 절대로 그리지 말 것.
4. 마치 잘 설계된 웹 대시보드 UI의 일부를 캡처한 것 같은 정제된 품질로 생성하라.
