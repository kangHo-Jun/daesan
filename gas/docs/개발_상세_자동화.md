# 개발_상세_자동화.md

## 0. 목적

구글 시트의 제품 데이터(제품명/설명/사진/섹션 타입)를 기반으로 **프롬프트 기반 이미지 생성(Prompt-Driven Generation)** 자동화 엔진을 구축한다.

- **Phase 1 (GPT)**: 제품 데이터를 분석하여 나노바나나 API가 즉시 렌더링할 수 있는 상세한 `full_image_prompt`를 생성한다.
- **Phase 2 (나노바나나)**: 생성된 프롬프트를 사용하여 800x1400 크기의 인포그래픽 이미지를 생성한다.
- **결과 기록**: 생성된 이미지 URL을 시트에 기록하고 진행 상태를 업데이트한다.

---

## 1. 운영 정책 (결정 사항)

### 1.1 생성 방식: 프롬프트 기반 (Prompt-Driven)
- 기존 템플릿(Modifications) 방식에서 **프롬프트 기반**으로 전환.
- GPT가 이미지의 레이아웃, 배경, 텍스트 배치(Section 1, 2, 3)를 모두 결정하여 프롬프트에 녹여낸다.

### 1.2 섹션 구성 및 데이터 활용
- **Section 1**: 헤더 및 제품 이미지. 시트의 `product_image_url`을 활용한다.
- **Section 2**: 핵심 스펙 (Comparison, Performance, Structure, Eco 중 택 1).
- **Section 3**: 주요 용도 및 특징 (3가지).

### 1.3 실행 및 동시성 제어
- **순차 처리**: 한 번의 실행으로 `READY` 상태인 모든 행을 순차적으로 처리한다.
- **LockService**: `withLock` 기능을 통해 동일 스크립트의 중복 실행을 방지한다. (최대 대기 180초)

## 2. 구글 시트 스키마 (컬럼 고정)

| 컬럼 | 이름 | 내용 |
|---|---|---|
| B | product_name | 제품명 |
| C | description_html | 제품설명(HTML) |
| D | product_image_url | 제품사진 URL |
| E | section2_type | comparison / performance / structure / eco |
| F | status | READY / PROCESSING / DONE / FAIL |
| G | result_image_url | 생성 결과 이미지 URL |
| H | last_error | 실패 사유 (에러 발생 시) |
| I | retry_count | 재시도 횟수 (최대 2회) |

**상태 전이**: `READY` → `PROCESSING` → `DONE` (성공) or `READY` (재시도) or `FAIL` (실패).

## 4. GPT 출력 스키마 (Strict JSON Mode)

GPT는 항상 `json_object` 모드로 동작하며, 아래 구조를 엄격히 준수한다.

```json
{
  "full_image_prompt": "이미지의 상세 레이아웃과 텍스트 내용을 포함한 영문 프롬프트..."
}
```

- **Temperature**: 0.1 (일관된 결과 보장)
- **Validation**: `full_image_prompt` 키가 없을 경우 에러 처리 및 재시도.

## 5. 모듈 구조 (GAS 파일 분리)

### 5.1 Code.gs (엔트리 및 메뉴)
- `onOpen()`: "Zart Image Engine" 커스텀 메뉴 생성.
- `runBatch()`: 전역 가시성 확보. `withLock`을 통해 `runBatchInternal` 실행.

### 5.2 batch.gs (배치 처리 핵심)
- `findReadyRows()`: `getActiveSheet()`를 사용하여 현재 활성화된 시트에서 데이터를 읽는다. 1행(헤더)을 제외하고 `READY` 상태인 행 번호를 수집하며, 공백 제거(`trim()`)를 적용한다.
- `processSingleRow(rowIndex)`: 개별 행 처리 및 상태 업데이트. 실패 시 `retry_count` 증가 및 상태 복구(`READY`) 또는 최종 실패(`FAIL`) 처리.
- `runBatchInternal()`: 전체 배치 실행 결과 집계 후 `SUCCESS`, `PARTIAL`, `FAIL` 중 하나를 반환한다.

### 5.3 gpt.gs
- `callGpt(rowData)`: 제품 데이터를 기반으로 `full_image_prompt`를 생성한다.

### 5.4 nano.gs
- `callNano(payload)`: 프롬프트를 사용하여 나노바나나 API의 `/render` 엔드포인트를 호출한다.

### 5.5 lock.gs
- `withLock(fn)`: `LockService`를 래핑하여 안전한 동시성 제어를 제공한다.

## 7. 운영 시나리오 및 체크리스트

1. **실행 전**: 시트의 `status`가 `READY`인지 확인하고, 설정된 API 키가 유효한지 확인한다.
2. **실행**: "Zart Image Engine" 메뉴에서 "Run Batch"를 클릭한다.
3. **결과 확인**:
   - `SUCCESS`: 모든 `READY` 행이 `DONE`으로 완료됨.
   - `PARTIAL`: 일부 행은 완료되었으나 일부 실패함.
   - `FAIL`: 모든 행이 실패하거나 시스템 오류 발생.
4. **문제 해결**: `last_error`(H열)를 확인하여 원인을 파악한다. (주로 API 할당량 소진 또는 프롬프트 거부 등)
